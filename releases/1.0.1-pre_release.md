# Summary

Fixed some bugs for the startup RAPID in-memory and recording diagnostics.

# What Changed

- 2 RAPID benchmark channels:
	- `node/rapid_memory_benchmark_operations_per_sec`
	- `node/rapid_recording_benchmark_channels_per_sec`
- Benchmark channels save the benchmark results if benchmarks are enabled in config upon engine startup.
- Used to calculate RAPID percent usage channels:
	- `node/rapid_memory_usage_percent`
	- `node/rapid_recording_usage_percent`

RAPID benchmark channels show the operator if they are close to maxing out the in-memory channel bandwidth, or the recording bandwidth on the machine that is running the engine.

Average in memory bandwidth is about `16 million ops/ second` (mix of read and write)

Average recording bandwidth is about `10 million channels / second` (without incurring a backlog)
