Tasks are the main way to package repeatable operator workflows into DARTWIC.

# What A Task Is

A task gives the operator a bounded action surface with clear inputs, state, and outcomes. It should help someone do one thing well, not replace the entire interface.

# Two Useful Shapes

- `task cards`
- `task configs`

# Good Task Design

- make the goal obvious
- show the minimum inputs needed
- surface validation early
- keep save/cancel behavior predictable

# Operator Notes

Operators should expect tasks to be deployment-specific. A good task feels like a focused workflow from the local team, not a generic control panel.
